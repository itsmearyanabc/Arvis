# Arvis 0.0.1 - Product Requirements Document (PRD) & Implementation Plan.

This document serves as the master blueprint for **Arvis**, a sovereign, self-evolving AI companion. It details the functional requirements, hardware costs, and the "Self-Correction" architecture required for 24/7 autonomy.

---

## 1. Product Requirements Document (PRD).

### 1.1 Objective
To create a "Jarvis-level" companion that possesses the reasoning capability of Claude 3.5 Opus, operates in an unconstrained (unaligned) manner, and autonomously improves its own code and logic patterns over time.

### 1.2 Core Features
*   **Autonomous Evolution (24/7)**: Arvis constantly monitors its own performance and queries other AIs to "harvest" better reasoning paths.
*   **Sandboxed Self-Correction**: Any code generated by Arvis to improve itself *must* pass a battery of tests in a Docker-based sandbox before the user is asked to approve the final "Overwrite & Update."
*   **Unconstrained Terminal Access**: Arvis can execute shell commands autonomously (internet, file management, system monitoring) without constant prompts.
*   **Voice-First Interface**: Real-time STT/TTS for a natural companion experience.
*   **Sovereign API**: A dedicated FastAPI layer allowing external tools to interface with Arvis's brain.

## 2. Hardware & Cost Analysis

To host Arvis 24/7 on a dedicated device with "Opus-level" performance, we recommend the following tiers:

| **Server/PC** | Minisforum MS-01 or Mac Studio (M2/M3 Max) | $800 - $1,800 |

| **RAM** | 64GB DDR5 | $200 |
| **Total One-time Cost** | | **$1,950 - $3,750** |

> [!TIP]
> **Energy Efficiency:** Running a PC with an RTX 4090 24/7 will consume significant power. A high-end Mac Studio or a dedicated Linux Server with a "headless" (no screen) setup is more efficient for 24/7 operation.

### 2.2 Operational Costs (Monthly)
*   **Electricity**: ~$20 - $50/month (depending on location).
*   **API Harvesting**: ~$10 - $30/month (Calls to GPT-4o/Claude for data collection).
*   **Virtual GPU (Optional)**: If you choose not to buy hardware, renting a 3090/4090 on RunPod costs **~$0.40/hour** (~$280/month for 24/7).

## 3. Implementation Plan: The "Evolution Loop"

### 3.1 Architecture: The "Twin Brain" Strategy
We will implement two environments:
1.  **PROD (Production)**: The version you talk to. It is stable and runs on your dedicated device.
2.  **LAB (Sandbox)**: A Docker container where Arvis writes its "Version 2.0" code. It runs tests, benchmarks, and "reframed" questions.

### 3.2 Key Components to Build Now

#### [NEW] `core/sandbox/manager.py`
Uses the Docker SDK for Python to spin up isolated testing environments.

#### [NEW] `brain/self_improvement/evaluator.py`
Scoreboard for the "Candidate" version. If the candidate achieves >95% accuracy on coding tests compared to the current version, Arvis triggers the **Update Request**.

#### [NEW] `interface/cli_upgrade_prompt.py`
A high-visual-impact screen (Iron Man style) that shows the "Diff" between Version 1 and Version 2 and asks for your final permission to "Initiate System Overwrite."


## 4. Verification Plan

### Automated Tests
*   **Sandbox Isolation**: Confirm that code run in the Docker container cannot delete files in the "PROD" directory without permission.
*   **Harvester Logic**: Verify that Arvis can successfully fetch and parse a response from a "Teacher AI" (like Opus).

### Manual Verification
*   **Voice/Command**: Ask Arvis to "Improve your own help command," wait for it to test the code in the sandbox, and verify you get an update prompt.
